# Chronicae for Windows: 재설계 로드맵 및 개발 지시서

## 1. 프로젝트 비전 및 목표

이 문서는 기존 macOS 전용 Chronicae 애플리케이션을 Windows 플랫폼에 맞춰 재설계하고 개발하기 위한 공식 로드맵입니다. 최종 목표는 Windows 사용자에게 macOS 버전과 동등하거나 그 이상의 가치를 제공하는 네이티브 애플리케이션을 제공하는 것입니다.

- **네이티브 경험:** WinUI3와 .NET MAUI를 기반으로 현대적인 Windows UI/UX를 제공합니다.
- **기능 동등성:** 노트 관리, 버전 기록, AI 기반 검색 등 macOS 버전의 모든 핵심 기능을 동일하게 지원합니다.
- **성능 최적화:** Windows 환경에 최적화된 성능과 안정성을 보장합니다.
- **코드 재사용:** 플랫폼에 독립적인 웹 기술(React SPA)과 API 명세는 최대한 재사용하여 개발 효율성을 높입니다.

## 2. 제안 기술 스택

| 구분 | 기술 | 사유 |
| --- | --- | --- |
| **데스크톱 UI** | **.NET MAUI (WinUI 3)** | Microsoft의 최신 UI 프레임워크로, Windows에 최적화된 네이티브 경험을 제공하며 C# 생태계와 완벽하게 통합됩니다. |
| **로컬 서버** | **ASP.NET Core Web API** | 고성능 C# 기반 서버 프레임워크로, 데스크톱 앱과 동일한 언어 스택을 사용하여 개발 일관성을 유지합니다. |
| **데이터베이스** | **SQLite 또는 LiteDB** | 별도의 설치가 필요 없는 가벼운 파일 기반 데이터베이스로, 로컬 애플리케이션에 적합합니다. |
| **웹 프론트엔드** | **React + Vite (기존 `vision-spa` 재사용)** | 플랫폼 독립적인 웹 기술 스택으로, 코드 변경 없이 Windows 버전의 내장 웹 뷰에서 그대로 호스팅합니다. |
| **언어** | **C# 12+** | .NET 생태계의 핵심 언어로, 현대적이고 강력한 기능을 제공합니다. |
| **패키지 관리** | **NuGet** | .NET 라이브러리 및 종속성 관리를 위한 표준 도구입니다. |

## 3. 개발 로드맵

### Phase 1: 기반 구축 및 핵심 기능 포팅 (1-2개월)

1.  **프로젝트 초기 설정:**
    -   Visual Studio 2022에서 신규 .NET MAUI 솔루션(`Chronicae.sln`)을 생성합니다.
    -   솔루션 내에 `.NET MAUI` 프로젝트(`Chronicae.Windows`)와 `ASP.NET Core Web API` 프로젝트(`Chronicae.Server.Windows`)를 추가합니다.
    -   기존 `vision-spa` 폴더는 솔루션과 동일한 레벨에 유지합니다.

2.  **서버 API 재구현:**
    -   `docs/api-spec.md`를 기준으로 `ASP.NET Core` 프로젝트에 REST/SSE 엔드포인트를 구현합니다.
    -   **우선순위:** `GET /status`, 프로젝트/노트 CRUD, `GET /events` (SSE) 등 핵심 API부터 구현합니다.
    -   SQLite 데이터베이스 연동 및 기본 데이터 모델(Project, Note)을 C# 코드로 정의합니다.

3.  **기본 UI 레이아웃 구현:**
    -   `.NET MAUI`를 사용하여 macOS 버전과 유사한 기본 앱 셸(사이드바, 콘텐츠 뷰)을 구성합니다.
    -   서버 시작/중지 로직을 구현하고 UI와 연동합니다.

4.  **핵심 기능 연동:**
    -   서버 API를 호출하여 프로젝트 및 노트 목록을 UI에 표시합니다.
    -   노트 선택 시 내용을 읽기 전용으로 표시하는 기능을 구현합니다.

### Phase 2: 기능 완성 및 고도화 (2-3개월)

1.  **전체 기능 구현:**
    -   노트 생성, 편집(Markdown 에디터 연동), 삭제 기능을 완성합니다.
    -   버전 기록 조회 및 특정 버전으로 복원하는 기능을 구현합니다.
    -   키워드 및 시맨틱 검색 API를 연동하고 UI를 구현합니다.
    -   설정 화면을 만들고, 포트 변경, 외부 접속 허용 등의 기능을 연동합니다.

2.  **Vision SPA 통합:**
    -   `Chronicae.Windows` 프로젝트의 빌드 프로세스에 `vision-spa`의 `npm run build` 스크립트를 연동합니다.
    -   빌드된 `dist` 폴더의 정적 파일들을 `ASP.NET Core` 서버가 `/web-app` 경로로 호스팅하도록 설정합니다.

3.  **AI 기능 통합:**
    -   AI 대화 관련 API(`POST /ai/query`) 및 SSE 스트림 로직을 구현합니다.
    -   필요시 C#에서 ONNX Runtime 등을 사용하여 로컬 RAG 파이프라인을 구축합니다.

4.  **Windows 특화 기능:**
    -   시스템 트레이 아이콘 및 컨텍스트 메뉴(열기, 종료 등)를 추가합니다.
    -   Windows 알림 센터를 통해 백업 완료, 인덱싱 상태 변경 등을 사용자에게 알립니다.

### Phase 3: 안정화 및 배포 (1-2개월)

1.  **품질 보증:**
    -   xUnit 또는 NUnit을 사용하여 핵심 로직에 대한 단위 테스트 및 통합 테스트를 작성합니다.
    -   UI 테스트 자동화를 위해 WinAppDriver 도입을 검토합니다.
    -   내부 테스트 및 사용자 피드백을 통해 버그를 수정하고 안정성을 확보합니다.

2.  **빌드/배포 자동화:**
    -   GitHub Actions를 사용하여 Windows 빌드, 테스트, 패키징(MSIX) 파이프라인을 구축합니다.
    -   Microsoft Store 또는 웹사이트를 통한 배포를 준비하고, 자동 업데이트 기능을 구현합니다.

3.  **문서화:**
    -   프로젝트 루트에 Windows 버전 개발을 위한 `README.md`를 새로 작성하거나 기존 문서를 업데이트합니다.
    -   사용자를 위한 도움말 및 가이드를 작성합니다.

## 4. 향후 지침

-   **API 우선 원칙:** 모든 기능 구현은 `docs/api-spec.md`에 정의된 API를 먼저 C# 서버로 구현한 후, UI와 연동하는 방식으로 진행합니다.
-   **플랫폼 간 일관성:** 핵심 데이터 모델과 비즈니스 로직은 최대한 플랫폼에 독립적으로 설계하여 향후 다른 플랫폼으로의 확장 가능성을 열어둡니다.
-   **점진적 개발:** 각 Phase의 목표를 달성할 때마다 안정적인 버전을 릴리스하여 점진적으로 프로젝트를 완성해 나갑니다.
