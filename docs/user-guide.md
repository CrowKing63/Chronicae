# Chronicae for Windows 사용자 가이드

## 소개

Chronicae for Windows는 노트를 효과적으로 관리하고 버전을 추적할 수 있는 강력한 노트 테이킹 애플리케이션입니다. 이 애플리케이션은 Windows 사용자에게 macOS 버전과 동일한 기능을 제공합니다.

## 설치 및 시작

### 시스템 요구 사항
- Windows 10 (19041.0 이상) 또는 Windows 11
- .NET 8 런타임 (앱과 함께 배포될 경우 생략 가능)

### 설치 방법
1. 공식 웹사이트에서 설치 파일 다운로드
2. 설치 파일 실행 및 설치 마법사 따르기
3. 설치 완료 후 애플리케이션 실행

## 시작하기

### 첫 실행
1. 애플리케이션 실행
2. "Start Server" 버튼 클릭하여 내부 서버 시작
3. 서버 상태가 "Running"으로 변경되면 사용 준비 완료

### 프로젝트 생성
1. "New Project Name" 입력란에 프로젝트 이름 입력
2. "Create Project" 버튼 클릭
3. 생성된 프로젝트가 프로젝트 목록에 표시됨

## 주요 기능

### 프로젝트 관리
- **프로젝트 생성**: 프로젝트 이름을 입력하고 "Create Project" 버튼 클릭
- **프로젝트 수정**: 프로젝트 목록에서 프로젝트 선택 후 이름 수정하고 "Edit Project" 버튼 클릭
- **프로젝트 삭제**: 프로젝트 목록에서 프로젝트 선택 후 "Delete Project" 버튼 클릭

### 노트 관리
- **노트 생성**: 프로젝트 선택 후 "New Note Title" 및 "New Note Excerpt" 입력, "Create Note" 버튼 클릭
- **노트 수정**: 노트 목록에서 노트 선택 후 제목/요약/내용 수정, "Edit Note" 버튼 클릭
- **노트 삭제**: 노트 목록에서 노트 선택 후 "Delete Note" 버튼 클릭

### 버전 관리
- **버전 확인**: 노트를 선택하면 해당 노트의 모든 버전을 확인할 수 있음
- **이전 버전 복원**: 특정 버전을 선택하고 복원 기능을 사용하면 이전 버전으로 노트를 복구할 수 있음

### 설정 관리
- **설정 화면 접근**: 메인 화면의 "Settings" 버튼 클릭
- **서버 포트 변경**: "Server Port" 필드에서 원하는 포트 번호 입력 후 "Save Settings" 클릭
- **외부 액세스 설정**: "Enable External Access" 스위치를 사용하여 외부 접속 허용 여부 설정
- **데이터 디렉토리 열기**: "Open Data Directory" 버튼을 클릭하여 애플리케이션 데이터 저장 위치 열기

## UI 요소

### 메인 화면
- **서버 상태 표시**: 서버 실행 여부 및 통계 정보 표시
- **시작/중지 버튼**: 내부 서버 시작 및 중지
- **프로젝트 목록**: 모든 프로젝트 표시
- **노트 목록**: 선택한 프로젝트의 노트 표시
- **설정 버튼**: 설정 화면으로 이동

### 설정 화면
- **서버 설정**: 서버 포트 및 외부 액세스 설정
- **정보 표시**: 애플리케이션 버전 및 데이터 디렉토리 위치 표시

## 자주 묻는 질문 (FAQ)

### Q: 서버가 시작되지 않아요. 어떻게 해야 하나요?
A: 다음 사항을 확인해 보세요:
- .NET 8 런타임이 설치되어 있는지 확인
- 지정한 포트가 다른 애플리케이션에 의해 사용 중인지 확인
- 방화벽 설정이 애플리케이션 실행을 차단하지 않는지 확인

### Q: 노트에 대한 버전 정보는 어디에서 확인할 수 있나요?
A: 현재 UI에서는 노트 버전 정보를 직접 표시하지 않습니다. 향후 업데이트에서 노트 버전을 확인할 수 있는 기능이 추가될 예정입니다.

### Q: 외부 액세스를 활성화하면 어떤 일이 일어나나요?
A: 외부 액세스를 활성화하면 로컬 네트워크의 다른 장치에서도 서버 API에 접근할 수 있습니다.

### Q: 데이터는 어디에 저장되나요?
A: 데이터는 SQLite 데이터베이스 파일로 사용자의 앱 데이터 디렉토리에 저장됩니다. 설정 화면의 "Open Data Directory" 버튼을 통해 위치를 확인할 수 있습니다.

## 문제 해결

### 일반적인 문제
1. **서버 시작 실패**: 포트가 이미 사용 중이거나, 필요한 런타임이 설치되지 않았을 수 있습니다.
2. **API 호출 실패**: 서버가 실행 중인지 확인하고, 올바른 포트를 사용하고 있는지 검증하세요.
3. **UI 반응 없음**: 애플리케이션이 응답하지 않는 경우, 프로세스를 종료하고 다시 시작해 보세요.

### 지원 요청
문제가 지속되거나, 기능 요청 또는 버그 신고는 공식 지원 채널을 통해 문의해 주세요.
